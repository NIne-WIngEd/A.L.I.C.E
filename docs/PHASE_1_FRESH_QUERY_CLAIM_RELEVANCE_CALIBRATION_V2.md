# Phase 1.11 — Fresh Query–Claim Relevance Calibration V2

The first relevance calibration was not representative:

- only eight eligible items were available;
- five of eight items came from one query;
- only one item was labeled fully relevant.

The cross-encoder also ranked a partially relevant claim above the fully
relevant claim, indicating that the MS MARCO reranker is more naturally suited
to broad query-relatedness than to the project's strict three-way direct-answer
rubric.

V2 therefore changes two things.

## Fresh candidate source

Candidates are generated by the current P1.11 pipeline after the Personal Claim
Validity Contract and existing evidence-support checks. This targets claims that
would actually survive the current system rather than recycling an older audit
pool.

Known regression queries remain excluded from threshold selection.

## Query diversity

Sampling has a hard per-query cap (default: 2). A single benchmark query cannot
dominate the calibration set.

## Dual evaluation objectives

The same human labels are evaluated in two ways:

### Strict relevance

- positive: relevant
- negative: partially_relevant, irrelevant

This tests whether the cross-encoder can act as a direct-answer acceptance gate.

### Non-irrelevance

- positive: relevant, partially_relevant
- negative: irrelevant

This tests whether the cross-encoder can act as a conservative rejection filter
for obviously unrelated claims.

No threshold selected here changes production behavior. Any promising threshold
must be frozen and evaluated on a fresh independent holdout.
